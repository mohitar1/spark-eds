  What the boilerplate should be

  A distribution-portal-in-a-box on the EDS + Cloudflare Worker stack,

  Ring 1 — The Core (every tenant gets this, untouched)

  is the heart of a distribution portal and is ~95% shared between the two;
  only UI/branding differs.

  Design rule for the core: zero tenant literals. Everything that says
  "santander" or "coke.com" or tccc: becomes a value read from tenant config.
  The core should be forkable into a new tenant with no edits to core files.

  Ring 2 — Optional Feature Modules (opt-in per tenant)

  This is where koassets is rich and Santander is bare. These should be
  detachable modules, not baked into core, so IBM can start minimal and grow:

  - Rights requests + review workflow (koassets my-rights-*, Fadel proxy,

  - Print jobs

  blocks, worker routes, and config. (This mirrors contenthub's

  Everything brand-specific, isolated so core stays pristine:

  becomes a per-tenant data file.

  toggling Ring-2 modules — never editing Ring-1.
